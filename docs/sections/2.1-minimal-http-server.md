# 2.1 - Minimal HTTP Server

## Overview

In this lesson, we transform `rustflow-server` from a `println!` binary into a real HTTP server using [Axum](https://github.com/tokio-rs/axum). By the end, you will have:

- Axum added as a workspace dependency
- A running HTTP server listening on port 3000
- Two endpoints: `GET /` and `GET /health`
- A clear understanding of every line of code involved

## What is Axum?

Axum is a web framework built on top of [Tower](https://github.com/tower-rs/tower) (a middleware framework) and [Hyper](https://github.com/hyperium/hyper) (an HTTP implementation), all running on [Tokio](https://github.com/tokio-rs/tokio) (an async runtime). This stack is known for being:

- **Ergonomic** — handler functions are just async functions with typed arguments
- **Composable** — middleware (called "layers") can be mixed and matched freely
- **Performant** — zero-cost abstractions; compiles down to highly optimized code
- **Type-safe** — the compiler catches routing and extraction errors at build time

The full stack looks like this:

```text
┌──────────────────────────┐
│       Your Handlers      │  ← You write these (async functions)
├──────────────────────────┤
│          Axum            │  ← Routing, extractors, responses
├──────────────────────────┤
│         Tower            │  ← Middleware (auth, logging, rate limiting)
├──────────────────────────┤
│         Hyper            │  ← HTTP/1.1 and HTTP/2 protocol handling
├──────────────────────────┤
│         Tokio            │  ← Async runtime (event loop, I/O, timers)
└──────────────────────────┘
```

You'll interact with all of these layers as the course progresses, but for now, we only need Axum and Tokio directly.

## Step 1: Add Axum to Workspace Dependencies

First, add `axum` to the workspace-level `Cargo.toml` so all crates can share the same version:

```toml name=Cargo.toml
[workspace]
resolver = "2"
members = [
    "crates/rustflow-server",
    "crates/rustflow-common",
]

[workspace.dependencies]
axum = "0.8"
tokio = { version = "1", features = ["full"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

Then reference it in `rustflow-server`:

```toml name=crates/rustflow-server/Cargo.toml
[package]
name = "rustflow-server"
version = "0.1.0"
edition = "2021"

[dependencies]
rustflow-common = { path = "../rustflow-common" }
axum = { workspace = true }
tokio = { workspace = true }
serde = { workspace = true }
serde_json = { workspace = true }
```

### Why not pin an exact version?

We use `"0.8"` rather than `"=0.8.1"` because Cargo's semver resolution will pick the latest compatible patch version. For a learning project this is fine. In production, you'd use `Cargo.lock` (which is committed for binaries) to ensure reproducible builds.

## Step 2: Write the Server

Replace the contents of `main.rs` with our first real HTTP server:

```rust name=crates/rustflow-server/src/main.rs
use axum::{routing::get, Router};
use rustflow_common::{APP_NAME, APP_VERSION, HealthStatus};
use tokio::net::TcpListener;

/// Root endpoint — a simple liveness message.
async fn root() -> &'static str {
    "RustFlow is running"
}

/// Health check endpoint — returns structured JSON about the service.
async fn health() -> axum::Json<HealthStatus> {
    axum::Json(HealthStatus::default())
}

#[tokio::main]
async fn main() {
    // Build the router with our two endpoints
    let app = Router::new()
        .route("/", get(root))
        .route("/health", get(health));

    // Bind to port 3000 on all interfaces
    let addr = "0.0.0.0:3000";
    let listener = TcpListener::bind(addr)
        .await
        .expect("Failed to bind to address");

    println!("{} v{} listening on {}", APP_NAME, APP_VERSION, addr);

    // Start serving requests
    axum::serve(listener, app)
        .await
        .expect("Server error");
}
```

That's it. Let's break down every piece.

## Anatomy of the Server

### The Router

```rust
let app = Router::new()
    .route("/", get(root))
    .route("/health", get(health));
```

A `Router` is Axum's routing table. Each `.route()` call maps a path to one or more HTTP method handlers. Here:

- `GET /` → calls the `root` function
- `GET /health` → calls the `health` function

The `get()` function is a shorthand for `MethodRouter::new().get(handler)`. Axum also provides `post()`, `put()`, `delete()`, `patch()`, etc. You can chain them:

```rust
// Example: same path, different methods (we'll use this later)
.route("/tasks", get(list_tasks).post(create_task))
```

### Handler Functions

```rust
async fn root() -> &'static str {
    "RustFlow is running"
}
```

An Axum handler is any async function that:
1. Takes zero or more **extractors** as arguments (we have none yet — covered in Section 2.3)
2. Returns something that implements **`IntoResponse`** (covered in Section 2.12)

Axum implements `IntoResponse` for many types out of the box:

| Return Type | HTTP Response |
|---|---|
| `&str` / `String` | `200 OK` with `text/plain` body |
| `axum::Json<T>` | `200 OK` with `application/json` body |
| `StatusCode` | That status code with an empty body |
| `(StatusCode, String)` | That status code with a text body |
| `Response<Body>` | The raw response as-is |

Our `root` handler returns `&'static str`, which becomes a `200 OK text/plain` response. Our `health` handler returns `Json<HealthStatus>`, which serializes the struct to JSON automatically.

### The Listener

```rust
let listener = TcpListener::bind(addr)
    .await
    .expect("Failed to bind to address");
```

We use `tokio::net::TcpListener` (not `std::net::TcpListener`) because we're in an async context. This binds a TCP socket to the given address. The `.await` is necessary because binding can fail asynchronously (e.g., port already in use).

> **Note:** `0.0.0.0` means "listen on all network interfaces." This is important for Docker later (Section 10.4). For local-only access, you could use `127.0.0.1`.

### Serving

```rust
axum::serve(listener, app)
    .await
    .expect("Server error");
```

`axum::serve` is the simplest way to run an Axum application. It takes a listener and a router, and runs forever (until the process is killed or an error occurs). The `.await` here blocks the `main` function — the server runs until interrupted.

### The Tokio Runtime

```rust
#[tokio::main]
async fn main() {
```

The `#[tokio::main]` macro transforms your `async fn main()` into a synchronous `fn main()` that creates a Tokio runtime and blocks on your async code. It roughly expands to:

```rust
fn main() {
    tokio::runtime::Builder::new_multi_thread()
        .enable_all()
        .build()
        .unwrap()
        .block_on(async {
            // your async main body here
        })
}
```

We specified `tokio = { features = ["full"] }` in our dependencies, which enables all Tokio features (timers, I/O, networking, macros, etc.). In production, you might slim this down — but for learning, `"full"` avoids confusing compilation errors.

## Step 3: Run and Test

### Start the server

```bash
cargo run -p rustflow-server
```

You should see:

```text
rustflow v0.1.0 listening on 0.0.0.0:3000
```

### Test with curl

Open a second terminal:

```bash
# Test the root endpoint
curl http://localhost:3000/
```

```text
RustFlow is running
```

```bash
# Test the health endpoint
curl http://localhost:3000/health
```

```json
{"service":"rustflow","version":"0.1.0","status":"ok"}
```

```bash
# Test with pretty-printed JSON
curl -s http://localhost:3000/health | jq
```

```json
{
  "service": "rustflow",
  "version": "0.1.0",
  "status": "ok"
}
```

```bash
# Verify that non-existent routes return 404
curl -i http://localhost:3000/nonexistent
```

```text
HTTP/1.1 404 Not Found
content-length: 0
...
```

Axum returns `404 Not Found` with an empty body for unmatched routes by default. We'll customize this in later sections.

### Test with a browser

Navigate to [http://localhost:3000](http://localhost:3000) — you should see "RustFlow is running" as plain text. Visit [http://localhost:3000/health](http://localhost:3000/health) to see the JSON response.

## What Just Happened?

Let's trace a request from curl to response:

```text
curl GET /health
      │
      ▼
┌─────────────────┐
│   TcpListener   │  Accepts the TCP connection on port 3000
├─────────────────┤
│      Hyper      │  Parses the raw bytes into an HTTP request
├─────────────────┤
│      Axum       │  Matches GET /health → health() handler
├─────────────────┤
│  health() fn    │  Creates HealthStatus::default(), wraps in Json()
├─────────────────┤
│   IntoResponse  │  Serializes to JSON, sets Content-Type header
├─────────────────┤
│      Hyper      │  Sends the HTTP response bytes back
└─────────────────┘
      │
      ▼
curl receives: {"service":"rustflow","version":"0.1.0","status":"ok"}
```
